# Comms

A record of the project's outbound community communications: announcements, invites, and calls for input. Normally this layer is invisible, handled quietly by whoever runs the community. Here it lives in the repo, so contributors can see what's been said, stay aligned on the messaging, and reuse what works.

## How it's organised

One file per announcement or campaign, dated: `YYYY-MM-DD-<slug>.md`. Each file holds the message variants (channel posts, DM templates) and a short status line on where and when it went out.

## What goes here, and what doesn't

This records the **outbound public messaging**: the posts and templates we send. It does not record people's private replies, who was messaged, or any personal data. Same sanitisation rule as the rest of the repo. If a reply turns into a research idea, it belongs in `context/ideas/`, not here.
